打造一个 Cloud Native 且 AI Native 的任务管理工具，从 CLI 入手是非常明智的工程决策。CLI 不仅开发迅速，而且其标准输入/输出（stdin/stdout）和结构化数据（如 JSON）天然适合作为 AI Agent 的外部工具（Tool Calling）被调用。

下面我们先梳理理论基础，然后直接进入 CLI 的接口设计。

---

### 一、 理论依据：艾森豪威尔矩阵的 AI Native 升级

传统的艾森豪威尔矩阵主要依靠**人工直觉**进行分类，这在信息爆炸的时代往往会成为认知负担。

* **Q1（重要且紧急）- 立即执行**：危机、即将到期的项目。
* **Q2（重要但不紧急）- 计划安排**：深度工作、系统架构设计、战略规划。这是高绩效个体的核心区域。
* **Q3（紧急但不重要）- 授权/自动化**：干扰、无意义的会议、重复性劳动。
* **Q4（不重要且不紧急）- 消除**：纯粹的消遣、时间黑洞。

**AI Native 的范式转移：**
相比于传统的线性清单系统，引入 AI Agent 后，矩阵的运作方式将发生根本改变：

1. **语义分类（Auto-Triage）**：你只需要输入“下周二前要把架构图发给团队”，AI 会自动提取时间（紧急度）和关键词（重要度），将其自动分配到 Q1 或 Q2。
2. **智能委派（Smart Delegation）**：对于 Q3（紧急不重要）的任务，AI Agent 不仅仅是将其标记，而是可以直接作为执行者。例如“帮我回绝明天的推销会议”，Agent 可以直接调用邮件 API 完成“委派”。
3. **动态优先级（Dynamic Prioritization）**：AI 可以根据你的上下文（当前时间、截止日期逼近程度、你的日历空闲状态）动态调整任务在象限中的位置。

### 二、 CLI 接口设计 (`tdmx` - Eisenhower Matrix CLI)

我们将这个 CLI 工具暂命名为 `tdmx`。为了更好地被大语言模型（如 Claude 或 OpenAI 的 API）调用，CLI 的设计必须兼顾人类易读性（Human-readable）和机器易读性（Machine-readable）。

#### 1. 核心命令与参数速查表

| 命令 (Command) | 说明 | 核心参数 (Options/Flags) |
| --- | --- | --- |
| `add` | 添加新任务 | `--auto`, `-q`, `-d`, `-p` |
| `ls` / `list` | 查看矩阵列表 | `-q`, `--json`, `--agent` |
| `update` | 更新任务状态 | `<id>`, `-q`, `--done` |
| `agent` | 唤起 AI 助手 | `--prompt`, `--interactive` |
| `sync` | 云端数据同步 | `--push`, `--pull` |

---

#### 2. 命令详细设计

**A. 添加任务 (`tdmx add`)**
最核心的输入接口，支持纯自然语言输入，让 AI 在后台处理结构化。

* **用法示例**：
* `tdmx add "修复生产环境数据库连接池泄漏" -q 1` (人类硬编码模式)
* `tdmx add "研究最新的向量数据库架构方案" --auto` (AI 自动分类模式)


* **参数**：
* `-q, --quadrant [1|2|3|4]`: 强制指定象限。
* `--auto`: **[AI Native 特性]** 将文本发送给内置轻量级模型，自动评估重要性和紧急性并分配象限。
* `-d, --due <datetime>`: 截止时间（支持自然语言如 "next friday"）。



**B. 视图检索 (`tdmx ls`)**
为了让 AI Agent 获取当前任务上下文，输出格式必须支持 JSON。

* **用法示例**：
* `tdmx ls -q 2` (仅查看重要不紧急的深度工作)
* `tdmx ls --json` (输出完整 JSON 数据，供外部 AI Agent 解析当前状态)


* **参数**：
* `-q, --quadrant`: 按象限过滤。
* `--json`: 以 JSON 格式输出，而非默认的终端表格渲染。



**C. 唤起智能代理 (`tdmx agent`)**
直接在终端中与任务数据库进行对话，利用 LLM 规划生活。

* **用法示例**：
* `tdmx agent "我今天只有下午 2 点到 4 点有空，我应该优先处理 Q2 中的哪两个任务？"`
* `tdmx agent "把 Q3 里所有需要发邮件的事情帮我起草好邮件草稿。"`


* **说明**：该命令会将本地的任务数据库（SQLite 或 JSON）作为 Context 注入到 Prompt 中，调用远端大模型生成规划或执行操作。

**D. 云端同步 (`tdmx sync`)**
作为 Cloud Native 应用的基石，确保本地 SQLite/JSON 文件能与远端保持一致。数据可以轻松同步至远端云服务（例如通过 AWS S3 或自定义的 Serverless 数据库），为未来跨设备（如 iOS 移动端）打下基础。

* **用法示例**：
* `tdmx sync --push`

### 三、 系统架构与 AI 交互流

当你或你的外部 AI Agent 执行 `tdmx add "规划新项目" --auto` 时，背后的数据流如下：

1. **CLI 解析**：捕捉到 `--auto` 标志。
2. **Prompt 组装**：系统生成隐藏 Prompt：*“评估任务：‘规划新项目’。根据艾森豪威尔矩阵，返回 JSON 格式：{"quadrant": 2, "reasoning": "战略规划，重要但不紧迫"}”。*
3. **API 调用**：CLI 请求云端 LLM。
4. **本地存储**：将解析后的结构化数据写入本地存储。
5. **云端同步**：后台静默执行云端备份。

这个 CLI 跑通之后，你不仅拥有了一个极客范的时间管理工具，而且它的底层 API 和数据库 schema 可以直接被未来的 iOS 版本复用。
